## Project

This project followed an Agile Scrum approach to develop features for a travel booking application, including destination lists, travel slideshows, user stories, testing artifacts, and project documentation.

## Reflection

### How do I interpret user needs and implement them into a program? How does creating user stories help with this?

I interpret user needs by gathering feedback from stakeholders and translating those requirements into specific features. User stories help organize requirements from the user's perspective and focus development on delivering value. During the SNHU Travel project, user stories helped identify features such as personalized recommendations, budget filters, and travel-type filters. The acceptance criteria associated with each story provided clear expectations for development and testing.

### How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach program development by first understanding the requirements, then breaking large tasks into smaller and more manageable pieces. Agile practices such as maintaining a product backlog, developing user stories, creating test cases, and gathering feedback throughout development help ensure that software meets user needs. In future projects, I hope to continue using iterative development, regular testing, sprint reviews, and retrospectives to improve quality and adaptability.

### What does it mean to be a good team member in software development?

Being a good team member means communicating effectively, collaborating with others, and remaining open to feedback and change. Throughout this course, I learned that successful software development depends on teamwork between Product Owners, Developers, Testers, and Scrum Masters. A good team member contributes their expertise, supports teammates, participates in discussions, and focuses on delivering value to users while helping the team achieve its goals.
